# キャパシティプランニング

- システムのリソースが将来的にも不足しないようにするための設計技術
    - CPU
    - メモリ
    - ディスク
    - ネットワーク帯域

## キャパシティプランニングとは

- 現実的にハードウェアにかけられるコストには限度がある
- なるべくコストを抑えつつ、システムの運用に支障が出ないようにする
    - システムが最大どれだけのリソースを必要とするか測定
    - その枠内に収まるようリソースを見積もる
- 困難性
    - ピーク時にリソースが大幅に不足
    - 現時点で十分に余裕があっても将来的に不足が常態化
        - 運用中に不可が増大していく
- 特に重要
    - CPU
    - メモリ
- 他にシステムパフォーマンスのボトルネックになるやつ
    - ディスクI/O
    - スワップの利用率
    - ネットワークI/O
- **パフォーマンスチューニング**
    - 効率よくシステム上のリソースを使えるよう、さまざまな設定を調整
